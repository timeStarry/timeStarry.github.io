使用 HTTPS 访问 GitHub Pages 站点

作者: 光溯星河 日期: 2023-01-12 阅读时间:0.9 分钟 字数:252 分类: 分类：技术博客 Github Pages 网络 博客搭建 Gridea 阅读量: -

博客搭建好之后，浏览器上的“不安全”三个字一直让我很难受。


于是我就去查了下关于Github pages静态网页的相关问题解决方案。发现放置网页的仓库设置>>Github Pages里提供了这么一个选项：

于是我就欣然勾上了，然后再一访问网页，出问题，网页样式没了。

然后到处查资料，得到了这些回答：

  • HTTPS地址中，如果出现了http资源，浏览器将认为这是不安全的，默认阻止资源的加载。
  • 访问css是绝对路径，需要改为https站点网址的相对路径
    ……
    看了一圈，终于这篇博客突然点醒了我，回头一看，我的Gridea远程设置依然是仓库地址

    将其修改为现在的域名timestarry.asia之后，同步，稍等一会儿，再次访问网页，发现网页样式显示恢复正常了，证书问题也解决了。
