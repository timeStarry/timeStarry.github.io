【面试题】记2023.03.04一些关于C语言的问答

作者: 光溯星河 日期: 2023-03-04 阅读时间:4.3 分钟 字数:1240 分类: C语言 面试 语言 分类：技术博客 随手记 阅读量: -

本文是今天六叔抽问的一些问题，关于C语言的散装知识，特此记录。

Q1：描述一下普通局部变量、普通全局变量、静态局部变量和静态全局变量的区别

Answer

  • 普通局部变量

    • 定义在函数内，仅能在所在函数内被访问
    • 空间分配在栈中
    • 生命周期：调用函数时诞生，函数执行完时销毁
    • 每次调用这个局部变量所在的函数时，这个局部变量在栈上的位置都不一定相同

  • 普通全局变量

    • 在函数外定义，可以被整个源程序访问
    • 放置于静态存储区中
    • 生命周期：定义时出生，随着程序结束而结束

  • 静态局部变量

    • 在函数内定义，但使用了static关键字，仅能在所在函数内被访问
    • 它们被编译器放在静态存储区中
    • 生命周期：定义时出生，随着程序结束而结束

  • 静态全局变量

    • 在函数外定义，并且使用了static关键字，只能被定义该变量的源文件内访问
    • 放置于静态存储区中
    • 生命周期：定义时出生，随着程序结束而结束

知识拓展：内存分区

上面提到了栈和全局储存区的概念，这也是一组很容易记着记着就记混了的概念，所以拓展一下，便于理解。

  • 常量区：用于存储常量数据，如字符串常量和浮点数常量。
  • 全局区：用于存储全局变量。
  • 静态区：用于存储静态变量。
  • 栈：用于临时存储数据，如函数参数、返回地址和局部变量。
  • 堆：用于动态分配内存。

Q2：哪些情况下会出现野指针

Answer

什么是野指针：指一个没有被初始化或已经释放的指针，它指向的内存地址是未知的或者是已经被释放的地址。

  1. 指针未初始化: 当指针变量被声明但未被初始化时，它会包含一个随机的内存地址，这个地址可能会是一个无效的地址。当尝试使用该指针变量时，程序会试图访问该地址，导致出现野指针。
  2. 指针指向已释放的内存: 当指针指向的内存已经被释放或者已经超出了它所属的作用域，那么指针就会变成野指针。
  3. 数组越界: 当使用指针访问数组元素时，如果超出了数组的范围，指针就会变成野指针。
  4. 返回局部变量的指针: 如果函数返回指向局部变量的指针，当函数结束时该局部变量的内存将被释放，指针将变成野指针。
  5. 动态分配内存后未正确释放: 当使用malloc、calloc或realloc等函数动态分配内存时，必须使用free函数释放内存。如果忘记释放内存，指针将变成野指针。

Q3：字节顺序：大端储存顺序、小端储存顺序

Answer

在大端字节顺序中，数据的高位字节被存储在低位地址，低位字节被存储在高位地址。

而在小端字节顺序中，数据的低位字节被存储在低位地址，高位字节被存储在高位地址。

举个例子，一个4字节的整数值0x12345678，在大端字节顺序中的存储顺序是：0x12 0x34 0x56 0x78，而在小端字节顺序中的存储顺序是：0x78 0x56 0x34 0x12。

知识拓展：在网络通信中的字节序

字节顺序的区别在于多字节数据的存储顺序，而对于单字节数据的存储顺序，两种字节顺序的结果是相同的。在网络通信中，为了保证通信的正确性，需要约定好数据的字节顺序。**常见的做法是使用网络字节顺序（Network Byte Order），即大端字节顺序作为统一的字节顺序。**因此，在进行网络通信时，需要将数据从主机字节顺序转换为网络字节顺序，通信结束后再将数据从网络字节顺序转换为主机字节顺序。常用的函数有htons、htonl、ntohs和ntohl等，用于进行字节顺序转换。

Q4：sizeof和strlen的区别

Answer

  • sizeof

    • sizeof是一个C语言运算符
    • 用于计算数据类型或变量在内存中所占的字节数
    • 注意，sizeof运算符的结果是一个常量表达式，可以在编译时计算出来
    sizeof(type)    // 计算数据类型type的大小
sizeof(expr)    // 计算表达式expr的大小

  • strlen()

    • strlen()是一个库函数
    • 用于计算字符串的长度，返回该字符串中字符的数量
    • 不包括字符串末尾的空字符('\0')
